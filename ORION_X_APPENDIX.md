```
1. IDENTIFIKATION
   ├─ Inbound: Webinar, Whitepaper, Website
   ├─ Outbound: LinkedIn Sales Navigator, Apollo.io

2. QUALIFIKATION (BANT)
   ├─ Budget: >€50k/Jahr

   ├─ Need: Aktuell 5+ Einzeltools im Einsatz

3. DEMO & POC
   ├─ 30-min Executive Demo (C-Level)
   ├─ 60-min Technical Deep Dive (IT)
   └─ 4-Wochen Proof of Concept (10 Nutzer)

4. VERHANDLUNG
   ├─ Standard: €5k–50k/Monat (je nach Nutzer)
   ├─ Custom: On-Premise, SLA, Support-Level

5. ONBOARDING (8 Wochen)
   ├─ Woche 1-2: Setup, Integrationen, Rollen
   ├─ Woche 3-4: Team-Training
   ├─ Woche 5-6: Erste Workflows, KI-Training
   └─ Woche 7-8: Review, Optimierung, Go-Live

6. EXPANSION
   ├─ Cross-Sell: Mehr Abteilungen
```

## ANHANG: GLOSSAR

| Begriff | Definition |
|---|---|
| **AI Agent** | Autonomer KI-Assistent mit spezifischer Rolle (Executive, Productivity, etc.) |
| **BFF (Backend for Frontend)** | API-Layer, der Frontend-spezifische Daten aggregiert |
| **Digital Twin** | Virtuelle Simulation eines Unternehmens/Prozesses |
| **Embedding** | Vektor-Darstellung von Text für semantische Suche |
| **Fine-Tuning** | Training eines Foundation Models auf spezifische Daten |
| **Headless API** | API-first-Architektur, die Frontend-unabhängig ist |
| **Knowledge Graph** | Netzwerk von Entitäten und Beziehungen (Neo4j) |
| **LLM (Large Language Model)** | Großes Sprachmodell (GPT-4o, Claude Opus) |
| **Memory System** | Mehrstufiges System zum Speichern und Abrufen von Kontext |
| **PWA (Progressive Web App)** | Web-App mit nativen Funktionen (Offline, Push) |
| **RAG (Retrieval-Augmented Generation)** | KI-Methode: Wissen aus DB + LLM-Response |
| **RBAC/ABAC** | Role-Based / Attribute-Based Access Control |
| **RSC (React Server Components)** | Serverseitig gerenderte React-Komponenten |
| **SSR (Server-Side Rendering)** | Serverseitiges Rendering für schnelle Ladezeiten |
| **TAM/SAM/SOM** | Marktgrößen: Total/Serviceable/Obtainable Market |
| **Vector Database** | Datenbank für Embeddings (semantische Ähnlichkeitssuche) |
| **WebAuthn** | Web Authentication API (Passkeys, Biometrie) |
| **Zero Trust** | Sicherheitsmodell: Nie vertrauen, immer prüfen |
